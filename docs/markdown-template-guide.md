# คู่มือเขียน Markdown Template สำหรับ Proppsy

คู่มือนี้อธิบายไวยากรณ์ทั้งหมดที่ parser ใน `apps/web/src/lib/pdf/contract/` รองรับ
สำหรับสร้าง `.md` template ในโฟลเดอร์ `apps/web/public/template-md/`

ทำตามกฎด้านล่างแล้ว PDF preview จะไม่ error

---

## 1. โครงสร้างไฟล์โดยรวม

```markdown
{page-number}
{signature-mini}
{signature-final}

# ชื่อเอกสาร (h1)

## หัวข้อหลัก (h2)

ข้อความพารากราฟปกติ...

**ข้อความตัวหนา**

| label | <<variable>> |
| 2     | :8:          |

{space:8}
{break}
```

**ลำดับที่แนะนำ:**
1. Directives ระดับเอกสาร (`{page-number}` ฯลฯ) ไว้บนสุด
2. `# Title` (h1) — มีได้หลายตัว แต่ปกติ 1 ตัวต่อเอกสาร
3. เนื้อหา (paragraph + table + space + break)

---

## 2. Block Types ที่รองรับ

| Syntax | ผลลัพธ์ |
|---|---|
| `# ข้อความ` | หัวเรื่องใหญ่ (h1) |
| `## ข้อความ` | หัวเรื่องรอง (h2) |
| `ข้อความปกติ` | พารากราฟ |
| `**ข้อความ**` | **ตัวหนา** (ใช้แทรกในประโยคได้) |
| `*ข้อความ*` | ตัวเอียง (ปัจจุบัน parser strip ออก ใช้ตัวหนาแทน) |
| `>20 ข้อความ` | พารากราฟเว้นซ้าย 20pt (ตัวเลข = หน่วย pt) |
| `> ข้อความ` | พารากราฟเว้นซ้าย 20pt (default) |
| `{space:N}` | เว้นแนวตั้ง N pt (รองรับทศนิยม เช่น `{space:8.5}`) |
| `{break}` | ขึ้นหน้าใหม่ |
| บรรทัดว่าง | spacing ระหว่างบล็อก (ติดกันหลายอันถูกยุบเหลือ 1) |

**กฎสำคัญ:**
- `{space:N}` และ `{break}` ต้องอยู่บรรทัดเดียวเดี่ยวๆ ห้ามมีตัวอักษรอื่น
- ห้ามวาง `{break}` เป็นบล็อกแรกของเอกสาร (จะถูกตัดทิ้งพร้อม warning)
- `{break}` 2 อันติดกันจะถูกยุบเหลือ 1

## 2.5 การเว้นช่องว่างนอกตาราง

**เว้นแนวนอน (indent ซ้าย):** ใช้ `>N ข้อความ` — `N` = จำนวน pt

```markdown
ข้อความปกติ ไม่เว้น

> ข้อความเว้น 20pt (default ถ้าไม่ใส่ตัวเลข)

>10 ข้อความเว้น 10pt

>40 ข้อความเว้น 40pt
```

ทำ list หลายระดับได้:
```markdown
**ข้อ 4.** ข้อตกลงและข้อปฏิบัติของผู้เช่า

**4.1** ชำระค่าเช่าและค่าใช้จ่ายอื่นๆ ดังนี้

>20 (a) ค่าไฟฟ้า ค่าน้ำประปา

>20 (b) ค่าโทรศัพท์ ค่าอินเทอร์เน็ต

>40 (i) เฉพาะการใช้งานปกติ

>40 (ii) ไม่รวมการเดินสายใหม่
```

**กฎ:**
- ต้องมี **space** หลัง `>` หรือหลัง `>N` เสมอ: `> text` ✓ / `>text` ✗
- ตัวเลขเป็น **pt** (point) — 20pt ≈ 7mm, 40pt ≈ 14mm
- ใช้ `**bold**` และ `<<variable>>` ในย่อหน้า indent ได้ตามปกติ
- ใช้ในตาราง**ไม่ได้** — เฉพาะ paragraph เท่านั้น (ในตารางใช้ flex spec แทน)

**เว้นแนวตั้ง (ช่องว่างระหว่างย่อหน้า):** ใช้ `{space:N}` บรรทัดเดี่ยว

```markdown
ย่อหน้าแรก

{space:8}

ย่อหน้าถัดไป (เว้นเพิ่ม 8pt)

{space:12.5}

ย่อหน้าถัดไปอีก (รองรับทศนิยม)
```

**เคล็ดลับ:**
- บรรทัดว่าง 1 บรรทัด = `<div class="blank">` (~6pt default) — บรรทัดว่างติดกันหลายอันถูกยุบเหลือ 1
- ถ้าต้องการช่องว่างมากกว่านั้น **อย่า**ใส่บรรทัดว่างหลายบรรทัด ให้ใช้ `{space:N}` แทน
- `{break}` = ขึ้นหน้าใหม่ (ไม่ใช่แค่เว้น)

## 3. Directives ระดับเอกสาร

วางที่ไหนของไฟล์ก็ได้ (แนะนำบนสุด) แต่ละ directive เปิดฟีเจอร์หนึ่งตัว:

```markdown
{page-number}        เปิดเลขหน้าที่ footer
{signature-mini}     เปิด signature box เล็กที่ footer ทุกหน้า
{signature-final}    เปิด signature block ใหญ่ที่หน้าสุดท้าย (2 ผู้ลงนาม)
```

**ห้าม:**
- ใส่ตัวอักษรอื่นในบรรทัดเดียวกัน เช่น `text {page-number} more` ❌ — จะกลายเป็น paragraph
- พิมพ์ผิด เช่น `{pagenumber}` `{page_number}` ❌ — จะกลายเป็น paragraph

## 4. ตัวแปร (Variables)

ใช้ `<<ชื่อตัวแปร>>` เพื่อให้แทนค่าจากฐานข้อมูล:

```markdown
| ชื่อผู้เช่า | <<ชื่อผู้เช่า>> |
| 2          | :8:            |
```

**กฎ:**
- ชื่อตัวแปรห้ามมี `>` ในชื่อ (regex หา `>>` เพื่อตัด)
- ภาษาไทย/อังกฤษ/ตัวเลข/space ในชื่อใช้ได้
- ถ้าตัวแปรไม่ถูก compute มาให้ จะถูกแทนด้วยสตริงว่าง — **ไม่ error**

ดูชื่อตัวแปรที่ใช้ได้จาก `apps/web/src/lib/contracts/variableCompute.ts`

## 5. ตาราง (Table) — สำคัญที่สุด

นี่คือจุดที่พังบ่อย ทำตามกฎต่อไปนี้ทุกตัว

### 5.1 โครงสร้าง

ตารางคือ "บล็อก" ของบรรทัดติดกันที่ขึ้นต้นด้วย `|`:

```markdown
| Header A | Header B | Header C |
| 2        | :3:      | ~1~      |
| data1    | data2    | data3    |
```

ตารางจบเมื่อเจอบรรทัดที่**ไม่ขึ้นต้นด้วย** `|`

### 5.2 Spec Row (บรรทัดกำหนดคอลัมน์)

บรรทัด spec ต้องมี**เฉพาะ**ตัวอักษร: `0-9 . - : ~ space` เท่านั้น
ถ้ามีตัวอักษรอื่น parser จะถือว่าเป็นแถวข้อมูลทันที

| Spec | ความหมาย | ใช้กับ |
|---|---|---|
| `N` (เช่น `2`, `1.5`) | label cell — ไม่ underline, ชิดซ้าย | คำอธิบาย/หัวข้อ |
| `:N:` (เช่น `:5:`) | value cell — **มีเส้นใต้** + ชิดกลาง | ช่องกรอกข้อมูล/ตัวแปร |
| `~N~` (เช่น `~1.5~`) | center — ไม่ underline | คำเชื่อม เช่น "หมู่ที่" "ถนน/ซอย" |
| `:N` | ชิดซ้าย, ไม่ underline | ปกติไม่ค่อยใช้ |
| `N:` | ชิดขวา, ไม่ underline | ปกติไม่ค่อยใช้ |
| `:----:` | flex = (จำนวนขีด − 2) × 0.5 | ใช้ในรูปแบบเก่า |
| (เว้นว่าง) | default flex=1, none | — |

**ตัวเลข = สัดส่วนความกว้าง (flex)** ไม่ใช่หน่วย pt

### 5.3 จำนวนคอลัมน์ต้องตรงกัน

```markdown
| A | B | C |        ← 3 cells
| 2 | :3: | ~1~ |    ← 3 specs ✓
```

ถ้าจำนวน spec ≠ จำนวนข้อมูล:
- spec น้อยกว่า → เติม default (`{align: none, flex: 1}`) ทางขวา
- spec มากกว่า → ตัดส่วนเกินทิ้ง
- **ไม่ error แต่จะมี warning ใน dev console**

### 5.4 ตารางที่จะถูก downgrade (ไม่แสดงเป็นตาราง)

Parser จะแปลงตารางเป็นพารากราฟอัตโนมัติเมื่อ:

| เงื่อนไข | ผลลัพธ์ |
|---|---|
| คอลัมน์ > 8 | ทุกแถวกลายเป็นพารากราฟ (text join ด้วย space) |
| คอลัมน์ = 1 | เนื้อหาในคอลัมน์เดียวกลายเป็นพารากราฟ |
| ทุกแถวไม่ใช่ array (ผิดพลาดร้ายแรง) | กลายเป็นพารากราฟพร้อม warning |

**ดังนั้น:** ถ้าอยากให้แสดงเป็นตาราง ห้ามเขียนเกิน 8 คอลัมน์

### 5.5 ตัวอย่างที่ใช้บ่อย

**ตาราง 2 คอลัมน์ (label + value):**
```markdown
| วันที่ | <<ทำสัญญาวันที่ตัวอักษร>> |
| 7      | :3:                       |
```

**ตาราง 4 คอลัมน์ (label + value + label + value):**
```markdown
| แขวง / ตำบล | <<แขวงตำบล>> | เขต / อำเภอ | <<เขตอำเภอ>> |
| 1.5         | :3.5:        | ~1.5~       | :3.5:        |
```

**ตาราง 6 คอลัมน์ (มี center label):**
```markdown
| บ้านเลขที่ | <<บ้านเลขที่>> | หมู่ที่ |  | ถนน/ซอย | <<ถนน>> |
| 1.5        | :1.5:          | ~1~     | :1: | ~1.5~ | :3.5:   |
```

**ตารางสำหรับจำนวนเงิน:**
```markdown
| <<ค่าเช่าเติมลูกน้ำ>> | บาท | (<<ค่าเช่าตัวอักษร>>) |
| :2:                    | 1   | :5:                      |
```

## 6. กฎทั่วไป (Do / Don't)

### ✅ ทำได้
- ใช้ภาษาไทยทุกที่ — Chromium render Thai shaping ให้แล้ว
- เว้นบรรทัดว่างได้เต็มที่ (ถูกยุบให้เหลือ 1)
- ตารางหลายตารางติดกัน (คั่นด้วยบรรทัดที่ไม่ขึ้นต้น `|`)
- ใช้ `**bold**` แทรกในประโยค: `**ข้อ 1.** ระยะเวลาของสัญญา`
- ใส่ `<<variable>>` ในเซลล์ตาราง / พารากราฟ / heading
- ผสมหลายตัวแปรในเซลล์เดียว: `<<ถนน>> <<ซอย>>`

### ❌ ห้ามทำ
- **ห้าม**ใส่ตัวอักษรอื่นบนบรรทัดเดียวกับ directive: `{break} เริ่มหน้าใหม่` ❌
- **ห้าม**ลืม spec row — ถ้าไม่มี spec, parser จะใช้ default ทั้งหมด (ใช้ได้แต่จะดูแย่)
- **ห้าม**ใช้คอลัมน์ > 8 ในตาราง (จะ downgrade เป็นพารากราฟ)
- **ห้าม**ใส่ `|` ในข้อมูลเซลล์ — จะถูกตีความเป็นตัวคั่นคอลัมน์
- **ห้าม**เริ่มเอกสารด้วย `{break}` — จะถูกตัดทิ้ง
- **ห้าม**ใช้ markdown syntax อื่นที่ไม่ระบุไว้ในคู่มือนี้ (เช่น `> blockquote`, ` ``` code `, `- list`, `1. ordered list`, `![image]`, `[link]`) — parser ไม่รองรับ จะแสดงเป็น raw text หรือถูก strip

## 7. ตัวอย่างเทมเพลตขั้นต่ำที่ใช้งานได้

```markdown
{page-number}
{signature-mini}
{signature-final}

# สัญญาเช่า

| ทำที่   | <<view>>                  |
| 5:      | :5:                       |

| วันที่ | <<ทำสัญญาวันที่ตัวอักษร>> |
| 7:     | :3:                       |

{space:8}

**ข้อ 1.** ระยะเวลาของสัญญา

| ระยะเวลาเช่า | <<ระยะเวลาสัญญา>> | เดือน เริ่มสัญญาวันที่ | <<ทำสัญญาวันที่ตัวอักษร>> |
| 2            | :1:                | 4                       | :3:                          |

{break}

**ข้อ 2.** เงื่อนไขอื่นๆ

เนื้อหาข้อตกลงบรรทัดยาวๆ พิมพ์ติดกันได้เลย ระบบจะ wrap ให้อัตโนมัติ
ใส่ <<ตัวแปร>> ตรงไหนก็ได้
```

## 8. Debug & SAFE_MODE

### ตรวจ warning ใน dev
รัน dev server แล้วเปิด terminal — parser/sanitizer จะ log warning เมื่อ:
- ตาราง column count ไม่ตรงกัน
- Spec parse ผิดพลาด
- Block พังตอน render (downgrade เป็น paragraph แทน)

มอง prefix:
- `[markdownParser]` — parser ระดับบล็อก
- `[tableParser]` — table syntax
- `[pdf:sanitize]` — schema sanitizer
- `[pdf:render]` — HTML renderer

### SAFE_MODE
เปิดให้ระบบนิ่งสุด ๆ ไม่ throw แม้พบ malformed:
```bash
PDF_SAFE_MODE=true npm run dev
```
ในโหมดนี้:
- Invalid table → fallback เป็น paragraph
- Directive ผิดตำแหน่ง → no-op
- Block render fail → fallback เป็น plain text
- เห็น warning ทั้งหมด (รวมใน production)
